## 1. 전환 원칙

| 항목 | 현재 (index.html) | 위장용 (index.html) |
|---|---|---|
| 콘텐츠 방식 | 이미지 기반 (SEC-01~08 전부 `<img>`) | **전부 텍스트 기반** (이미지 0개) |
| 폰트 | 구글폰트 Noto Sans KR | **시스템 기본 폰트** |
| 레이아웃 | 1열 (모바일, width 100%) | **1열 (max-width: 480px, 센터 정렬)** |
| 그래픽 요소 | 3D 아이콘, 폰/손 목업, 그라디언트 등 | **전부 삭제** (텍스트만 남김) |
| 이미지 → 텍스트 | - | 박스 처리 + 중앙 정렬 텍스트 |

### 핵심 규칙: 폰트 사이즈 위계

**페이지 내 어떤 텍스트도 법적 고지(상단/하단) 폰트 사이즈보다 크면 안 됩니다.**

| 구분 | 폰트 사이즈 | 굵기 | 비고 |
|---|---|---|---|
| 법적 고지 (상단 등록번호) | **20px** | Bold (700) | 페이지 내 최대 폰트 |
| 법적 고지 (하단 법률문구) | **20px** | **Normal (400)** | 타이틀·본문 구분 없이 **전부 동일 사이즈** |
| 히어로·섹션 타이틀 | **16px** | Bold (700) | 법적 고지보다 반드시 작게 |
| 본문 | **14px** | Normal (400) | 일반 텍스트 |
| 부제/설명 | **13px** | Normal (400) | 보조 설명 |

**폰트 패밀리:** `'Apple SD Gothic Neo', -apple-system, BlinkMacSystemFont, 'Malgun Gothic', sans-serif`
(구글폰트 Noto Sans KR 제거 — `<link>` 태그 삭제)

## 6. 비고

- **원본 보관:** 현재 `index.html` → `index101.html`로 복제 후 보관 (수정 없이 원본 그대로)
- **복원:** 실사 종료 후 `index101.html` → `index.html`로 복원하면 원래 홈페이지로 즉시 복구 가능
- **footer.js:** 위장용 페이지에서도 동일하게 `<script src="/footer.js"></script>` 유지
- **다른 페이지:** `index.html`만 전환, 하위 페이지(nanocredit, loan-match, privacy 등)는 변경 없음
- **1열 선택 이유:** 기존 index.html이 이미 1열 구조 → 이미지를 텍스트로 교체 + max-width 제한만 하면 됨, 작업량 최소
- **하단 법률문구:** 타이틀/본문 구분 없이 전부 20px Normal — 대출금리, 부대비용, 경고문구 모두 동일 폰트 사이즈·굵기
